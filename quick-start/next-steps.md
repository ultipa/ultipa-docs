# 6. Next Steps

You have installed GQLDB, loaded a graph, queried and traversed it, run an algorithm, and driven it all from an AI agent. Here is where to go deeper on each thread.

## Go Deeper on the Query Language

- <a href="/docs/gql/introduction" target="_blank">ISO GQL</a>: the full language reference: pattern matching, data manipulation, functions, expressions, and transactions.
- <a href="/docs/stored-procedures/quick-start" target="_blank">Stored Procedures</a>: package reusable logic you can call from your queries.

## Model Your Data Properly

- <a href="/docs/gql/open-graphs" target="_blank">Open Graphs</a>: the schema-free default used in this guide, where labels and properties are created on the fly.
- <a href="/docs/gql/closed-graphs" target="_blank">Closed Graphs</a>: enforce a schema of node and edge types for production data integrity.
- <a href="/docs/ontology/introduction" target="_blank">Ontology</a>: model RDF data with OWL semantics, inference, and validation.

- <a href="/docs/ai-and-vectors/overview" target="_blank">AI & Vectors</a>: vector search and AI-powered graph functions.
