# Introduction

## What is UQL

**UQL** (Ultipa Query Language) is a declarative, high-performance language specifically designed for querying and managing data in the Ultipa Graph Database. UQL enables intuitive and efficient interaction with graph-structured information, optimizing both usability and performance.

You can use UQL statements to create, update, delete, and retrieve data from a graph. UQL also provides advanced capabilities for managing graph structures, optimizing database performance, and performing graph analytics. With its expressive yet user-friendly syntax, UQL enables complex traversals and in-depth analysis with minimal code and a low learning curve.

## How to Run UQL

UQL can be executed within the Ultipa ecosystem through the following methods:

- <a target="_blank" href="/docs/tools/cli">Ultipa CLI</a>: A command line interface for executing queries.
- <a target="_blank" href="/docs/manager-user-guide">Ultipa Manager</a>: A web-based, highly visualized graph query and management platform.
- <a target="_blank" href="/docs/drivers">Ultipa Drivers</a>: Official libraries for interacting with Ultipa graph databases in various programming languages, along with a RESTful API.
- <a target="_blank" href="https://youtu.be/97lxSg_fsuc?si=fW67u6UurgNw6f6u">Visual Studio Code</a>: An integrated development environment (IDE) where UQL queries can be executed using the dedicated extension.

## Core Capabilities of UQL

Like most database languages, UQL provides comprehensive support for DQL, DDL, DML, and DCL operations:

- **DQL** (Data Query Language): Retrieve data such as nodes, edges and paths from the graph database.
- **DDL** (Data Definition Language): Define the graph structure (schema and properties), create indexes, and manage other structural elements.
- **DML** (Data Manipulation Language): Add, modify, and delete data within the graph database.
- **DCL** (Data Control Language): Manage access to the database and its objects, granting or revoking permissions for specific operations.
